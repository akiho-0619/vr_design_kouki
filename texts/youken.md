 ** プロジェクト概要 ** 
- 目的: easythree.jsを用いて「箱庭環境」を構築し、複数のエージェントを配置。エージェントはパラメータに基づいて行動を変化させ、行動遷移はマルコフ連鎖で決定する。
- 成果物: Webブラウザ上で動作するインタラクティブなシミュレーション環境。

#機能要件
## 箱庭環境
- easythree.jsで地形・背景・オブジェクトを描画。
- 環境オブジェクト例:
- 地形: 草原、森、池、岩場
- 資源: 木、岩、食料
- 交流ポイント: ベンチ、焚き火、広場
- 障害物: 壁、川、崖
- 環境の広さ：初期値は固定、min <= ユーザー定義 <= maxで変更可能
- 環境の外部要件：昼夜サイクルや天候変化は環境との相互作用があればつける

## エージェント
- 複数体生成（数は要検討: 10体程度？）
- 表現方法:VRMモデルをエージェントとする
- cube（抽象オブジェクト）
 -- cubeの方が色や大きさでいろいろ視覚化できるように

   +----------+
   |          |
 +-+          +-+

   |          |
   +----------+
      | |  | |
      +-+  +-+

## パラメータ
- 各エージェントに以下のパラメータを持たせる:
- 好奇心（探索範囲）
- 社交性（他エージェントとの交流頻度）
- エネルギー（活動時間と休憩時間）
- 速度（移動補間の速さ）
- 攻撃性（他エージェントとの競合確率）
- 協調性（グループ行動確率）
- 初期値: ランダム or 1 <= value <= 100のユーザー定義

## 行動一覧
- 探索: ランダムな目的地へ移動
- 交流: 他エージェントに接近しジェスチャー
- 休憩: 停止・座る・寝る
- 採取: 資源オブジェクトに接近し取得
- 攻撃/防御: 他エージェントと競合
- 協調: グループで同じ方向へ移動
- 行動モーション：アニメーションがあれば導入する

## 行動遷移（マルコフ連鎖）
- 状態集合: {探索, 交流, 休憩, 採取, 攻撃, 協調}
- 遷移確率行列を定義し、パラメータによって動的に変更。
- 例: 探索中 → 交流 (0.2), 休憩 (0.3), 採取 (0.5)
- 遷移確率: 遷移確率行列はの初期値は固定、ユーザー定義可能とする

# 非機能要件
